  • One/Two columns: as your wish!
  • Widgets areas: there's a lot of space for your favorite widgets
  • Translation ready: Shiword is ready to be translated in your language
  • Two menus: one below the header and one above the footer
  • Standard WordPress: fully compliant with the standards required by WordPress
  • Featured images
  • Gravatars Support: your gravatar follows you everywhere
  • Quickbar navigation: all the most useful links always at your disposal
  • Post/Pages slideshow: make your most interesting post/pages more visible
  • Print preview: you want to print an article? look at the preview first!
  • Menu animations: jquery powered
  • Custom header: you are your blog!
  • Custom colors: paint your site with the skin builder tool
  • Custom widgets : lots of new widgets at your disposal
  • Mobile support : Shiword has a dedicated style for mobile view (touchscreen friendly)
  • Post formats: support for post formats ("aside", "gallery", "audio", "quote", "image", "video", "link", "status")
  • Custom Template "One column, no sidebar": a custom page template without sidebar

TECHNICAL AND LIMITATIONS:

  • *.dev.js files are the same as *.min.js, but commented and uncompressed
  • theme uses jQuery javascript framework
  • if you experience any javascript problem, try disable animations in Appearance -> Theme options
  • print_preview.css is a stylesheet file used to emulate a print preview.
  • in order to use the new widgets, you need to activate them in Appearance -> Widgets
  • the secondary menu (above the footer widget area) support only one level of hierarchy
  • the theme detects the following mobile devices/browsers/os : ipod, iphone, android, opera mini, blackberry, palm, symbian and many more
  • "enhanced post title" option. Shows the post title with the featured image only if the image is big enough (min 700x200 px)
  • mobile view doesn't support custom templates

POST FORMATS:

Shiword support post formats, the new feature introduced by WP3.1. The theme uses a special style for every format in index view, while, in single view, it will always display the whole content. In order to correctly use post formats, you must follow these guidelines

  • aside: nothing special to do here
  • gallery: the code will search for the first gallery shortcode inside the post content. Otherwise it will search every image attached to post.
  • audio: the code will search for all the linked mp3's inside the post content. Make sure there's one, at least
  • quote: the code will search for the first <blockquote> tag inside the post content. Otherwise, the whole content will be wrapped inside a blockquote tag
  • image: the code will search for the first <img> tag inside the post content. Make sure there's one, at least
  • video: nothing special to do here
  • link: the code will search for the first <a> tag (anchor) inside the post content. Make sure there's one, at least
  • status: nothing special to do here
  • chat: nothing special to do here
Remember to set the desired format to each post in edit page.
